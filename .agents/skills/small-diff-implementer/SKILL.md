---
name: small-diff-implementer
description: Implement small, focused changes with minimal risk and clear tests.
---

**Trigger:** Any implementation request  
**Inputs:** task + touched subsystem  
**Steps:**
1. Identify minimal file set
2. Implement smallest viable change
3. Avoid unrelated refactor
4. Add/adjust focused tests
**Output:** compact diff  
**Done when:** change is understandable in one reading pass.
